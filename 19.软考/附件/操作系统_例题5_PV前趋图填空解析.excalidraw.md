## Text Elements
PV 前趋图填空，核心是“前驱结束 V，后继开始 P”
通用规则：一条前趋边对应一个信号量。
如果 A -> B，那么 A 执行完要 V(Sx)，B 开始前要 P(Sx)。 ^ruleText

P1 ^P1t

P2 ^P2t

P3 ^P3t

P4 ^P4t

P5 ^P5t

P6 ^P6t

S1 ^tS1

S2 ^tS2

S3 ^tS3

S4 ^tS4

S5 ^tS5

S6 ^tS6

S7 ^tS7

S8 ^tS8

第 1 组：① 和 ②

P1 后面连到 P2、P3
所以 ① = V(S1)、V(S2)
P2 进入前只需等 P1
所以 ② = P(S1) ^step1t

第 2 组：③ 和 ④

P3 前面有两条边：P1 -> P3、P2 -> P3
代码里已经写了 P(S2)
所以缺的 ③ = P(S3)
P3 执行完后要放行 P4、P5
所以 ④ = V(S5)、V(S6) ^step2t

第 3 组：⑤ 和 ⑥

P5 代码里已有 P(S6) ... V(S8)
说明 P3 -> P5、P5 -> P6 已处理
所以 P4 执行完应补 ⑤ = V(S7)
P6 必须同时等 P4、P5
所以 ⑥ = P(S7)、P(S8) ^step3t

最终答案

① = V(S1), V(S2)
② = P(S1)
③ = P(S3)
④ = V(S5), V(S6)
⑤ = V(S7)
⑥ = P(S7), P(S8)

对应选项：C / B / D ^anst
