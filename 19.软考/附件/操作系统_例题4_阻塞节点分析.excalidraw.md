## Text Elements
P1 是阻塞节点

P1 当前在请求 R2。
R2 一共有 3 个实例，图中已经全部分配出去，
当前没有空闲 R2 可再分给 P1。

因此：P1 的请求不能立即满足 -> P1 阻塞。 ^y9En2FyS

P2 是阻塞节点

P2 当前在请求 R1。
R1 一共有 2 个实例，图中这 2 个实例都已分配，
当前没有空闲 R1 可再分给 P2。

因此：P2 的请求不能立即满足 -> P2 阻塞。 ^dz9THoXw

P3 是非阻塞节点

P3 当前在请求 R3。
R3 一共有 2 个实例，图中只分配出 1 个，
说明当前还剩 1 个空闲 R3。

因此：P3 的请求可以立即满足 -> P3 非阻塞。 ^isi9CRvL

化简顺序

1. 先看出 P3 是非阻塞节点，因此先化简 P3。
2. 回收 P3 已占有的资源，R2 和 R1 系统空闲资源增加，此时 P1 变为非阻塞节点。
3. 再化简 P1，继续回收资源。
4. 最后 P2 也变为非阻塞节点，因此再化简 P2。

所以资源图可以化简，化简顺序为：P3 -> P1 -> P2。

为什么 P3 资源释放后，先化简的 P1 而不是 P2 呢？
P3 完成并释放资源后，P1 和 P2 都会变成非阻塞节点，谁先化简都可以。题目给出的选项里只有 P3 -> P1 -> P2 这个可行顺序，所以标准答案选了它。 ^x64AkZR5
