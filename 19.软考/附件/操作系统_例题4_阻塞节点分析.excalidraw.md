## Text Elements
P1 是阻塞节点

P1 当前在请求 R2。
R2 一共有 3 个实例，图中已经全部分配出去，
当前没有空闲 R2 可再分给 P1。

因此：P1 的请求不能立即满足 -> P1 阻塞。 ^text_p1

P2 是阻塞节点

P2 当前在请求 R1。
R1 一共有 2 个实例，图中这 2 个实例都已分配，
当前没有空闲 R1 可再分给 P2。

因此：P2 的请求不能立即满足 -> P2 阻塞。 ^text_p2

P3 是非阻塞节点

P3 当前在请求 R3。
R3 一共有 2 个实例，图中只分配出 1 个，
说明当前还剩 1 个空闲 R3。

因此：P3 的请求可以立即满足 -> P3 非阻塞。 ^text_p3

结论：判断阻塞节点时，不是先看有没有环，而是先看“当前请求能否由空闲资源立刻满足”。因此 P1、P2 阻塞，P3 非阻塞。 ^yTMUYCEj

化简顺序

1. 先看出 P3 是非阻塞节点，因此先化简 P3。
2. 回收 P3 已占有的资源后，系统空闲资源增加，此时 P1 变为非阻塞节点。
3. 再化简 P1，继续回收资源。
4. 最后 P2 也变为非阻塞节点，因此再化简 P2。

所以资源图可以化简，化简顺序为：P3 -> P1 -> P2。 ^lyKD3Lkf
