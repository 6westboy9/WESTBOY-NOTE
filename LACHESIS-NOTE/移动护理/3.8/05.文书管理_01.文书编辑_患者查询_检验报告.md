# 获取数据

获取数据的来源有两种：

* 直查（实时调用同步程序的接口获取HIS的检验数据）
* 数据库（已经同步到数据库了，直接从数据库查询）


## 1.确认是否开启直查

**第一种方式：直接查看配置**


>配置路径：`usr/local/springboot/springbootX_emr/WRMSEmr/config/sync.config`

1. `direct-search.direct-search=1` 说明开启了直查总开关
2. 但是检验是否走直查，还需要确认有没有配置以下两个方法，两个是独立的，配置了哪个对应底层逻辑就走直查。
	* 查询检验主数据： `com.lachesis.windranger.emr.dao.PatLabMasterMapperExt.selectByPidOrInhosCodeWithDate` 
	* 查询检验明细数据：`com.lachesis.windranger.emr.dao.PatLabItemMapper.selectByBean`

```properties
direct-search.direct-search=1
direct-search.external-db-query-url:http://127.0.0.1:9098/data-sync/direct
direct-search.use-external-db-ids=\
  #com.lachesis.windranger.emr.dao.PatExamMasterMapper.selectByBean,\
  #com.lachesis.windranger.emr.dao.PatExamMasterMapperExt.selectExamMasterByCondition,\
  #com.lachesis.windranger.emr.dao.PatExamMasterMapperExt.selectExamListByPidOrInhosCodeWithDate,\
  com.lachesis.windranger.emr.dao.PatExamReportMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatExamReportMapperExt.selectByBean,\
  com.lachesis.windranger.emr.dao.PatLabMasterMapperExt.selectByPidOrInhosCodeWithDate,\
  com.lachesis.windranger.emr.dao.PatLabItemMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatInhosEmrdocumentMapperExt.getRecordsByCondition,\
  com.lachesis.windranger.emr.dao.PatInhosEmrdocumentMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatInhosFeeDetailMapperExt.getFeeByInhosCodeAndDate,\
  com.lachesis.windranger.emr.dao.PatInhosFeeDetailMapperExt.getFeeByInhosCodesAndDate,\
  com.lachesis.windranger.emr.dao.PatInhosFeeDetailMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatInhosFeeSummaryMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatInhosPayRecordMapper.selectByBean,\
  com.lachesis.windranger.emr.dao.PatInhosPrePayHistoryMapper.selectByBean
```

**第二种方式：查看日志**

在请求时已经获取到了接口调用数据，使用tlogtraceid在日志文件中进行搜索：

日志路径：`usr/local/springboot/springbootX_emr/WRMSEmr/LOGS/`

```
grep "tlogtraceid" log_info.log
```

如果查看到“直查请求”这四个字，说明走的是直查逻辑。

同时可以通过”直查结果“查看直查的数据结果。

## 2.直查逻辑

核心逻辑在同步程序侧，同步程序提供直查的`HTTP`接口，`emr`系统只负责`HTTP`接口调用。

## 3.非直查逻辑

可以根据SQL先初步确认下**检验主数据**有无：

```SQL
-- 11795817 ➡️ curl参数上的pid
-- 11795817 ➡️ curl参数inhosCode
-- 2026-03-06 00:00:00 ➡️ 调用接口的requestBegin，格式为加了时分秒：00:00:00
-- 2026-03-18 00:00:00 ➡️ 调用接口的requestEnd+1天，格式为加了时分秒：00:00:00
SELECT *
FROM pat_lab_master m
WHERE (m.pat_code = '11795817' OR m.inhos_code = '25476776') 
  AND m.requested_date_time >= '2026-03-06 00:00:00'
  AND m.requested_date_time <= '2026-03-18 00:00:00'
```
