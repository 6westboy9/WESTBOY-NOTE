
执行`free -h`命令查看Linux系统内存和Swap使用情况的结果。各字段含义如下：

![[Pasted image 20250627132049.png|L|700]]

### 第一部分：物理内存（`Mem:`）

| 字段         | 含义                 |
| ---------- | ------------------ |
| total      | 总物理内存（31G）         |
| used       | 已用内存（26G）          |
| free       | 空闲内存（1.1G）         |
| shared     | 多个进程共享的内存（1.6G）    |
| buff/cache | 缓冲/缓存使用的内存（3.8G）   |
| available  | 可用内存（不影响应用使用，3.0G） |

> **注意：**`used`并不等于内存紧张，因为`buff/cache`也是Linux出于性能优化预留的缓存，一旦有应用需要，可以马上释放。

### 第二部分：交换空间（`Swap:`）

| 字段    | 含义           |
| ----- | ------------ |
| total | 总交换空间：4.0G   |
| used  | 已使用交换空间：4.0G |
| free  | 剩余交换空间：132K  |

> **关键问题：Swap使用完了！**
> 
> - 说明物理内存已经不够用，系统开始大量使用Swap。
> - 当前Swap几乎被用尽，这通常意味着内存压力很大，甚至会导致系统性能下降或触发OOM Killer。

### 总结

你的系统当前内存使用情况如下：

- **物理内存（RAM）31G，已使用26G，只剩1.1G 空闲，实际可用仅3G**；
- **Swap（虚拟内存）4G 已全部用完，仅剩132K**；
- **系统内存已经接近耗尽，存在内存压力风险，建议排查大内存进程或考虑扩容内存**。
