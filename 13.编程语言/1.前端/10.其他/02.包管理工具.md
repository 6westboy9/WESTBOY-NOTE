# npm - 传统老大哥

npm是node.js默认的包管理系统，**在安装node的时候会一起安装**，用于管理node中的第三方插件。

## npm与npx

[[02.包管理工具_npm与npx]]

|功能|`npm`|`npx`|
|:--|:--|:--|
|安装包|✅ `npm install`|❌ 不安装|
|全局运行命令|✅ 需先 `npm install -g`|✅ 直接运行|
|临时运行命令|❌ 不支持|✅ 支持|
|避免污染全局环境|❌ 会安装到全局|✅ 不污染全局|

## npm镜像源切换

```sh
npm config get registry # 查看当前镜像
npm install -g nrm
nrm ls            # 查看可用镜像
  npm ---------- https://registry.npmjs.org/
  yarn --------- https://registry.yarnpkg.com/
  tencent ------ https://mirrors.tencent.com/npm/
  cnpm --------- https://r.cnpmjs.org/
* taobao ------- https://registry.npmmirror.com/
  npmMirror ---- https://skimdb.npmjs.com/registry/
  huawei ------- https://repo.huaweicloud.com/repository/npm/
nrm use taobao   # 切换到淘宝源
```

## npm多版本切换

nvm即`node version manager`，好处是方便切换node.js的版本。

安装注意事项

1. 要卸载现有的node.js
2. 提示选择nvm和node.js目录时，一定要<font color="#f79646">避免目录中出现空格</font>
3. 选用<font color="#f79646">以管理员身份运行cmd程序</font>，在cmd程序中执行nvm的命令
4. 首次运行前设置好国内镜像地址

```
nvm node_mirror http://npm.taobao.org/mirrors/node
nvm npm_mirror http://npm.taobao.org/mirrors/npm
```

# pnmp vs npm **🚀 推荐使用**

|对比项|npm|pnpm|
|:--|:--|:--|
|定位|Node 官方默认包管理器|高性能、高效 npm 替代方案|
|依赖结构|扁平化，易产生**幽灵依赖**|非扁平化 + 链接，**无幽灵依赖**|
|磁盘占用|高，多项目重复存储|极低，全局一份，多项目共享|
|安装速度|普通，缓存后提升有限|极快，缓存后接近秒装|
|依赖稳定性|一般，易版本冲突|高，版本确定性强|
|monorepo|支持弱，需额外配置|原生支持，业界主流|
|兼容性|100% 兼容所有包、老环境|100% 兼容 npm 生态|
|学习成本|零，开箱即用|极低，命令基本一致|
|适合场景|新手、简单项目、老旧项目|中大型项目、团队、多项目|
