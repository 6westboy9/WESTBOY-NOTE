# 层叠

## 样式来源

| 来源              | 通俗解释                      | 优先级 |
| --------------- | ------------------------- | --- |
| 作者的`!important` | 程序员写的带有`!important`的CSS样式 | 1   |
| 作者              | 程序员写的不带`!important`的CSS样式 | 2   |
| 用户定义一个用户样式表     | 很少用                       | 3   |
| 用户代理            | 浏览器默认样式                   | 4   |
> [!tip] 提示
> 有些浏览器允许用户定义一个用户样式表。它的优先级介于用户代理样式表和作者样式表之间。用户样式表很少见，并且不受网站作者控制，因此这里略过。

## 理解优先级

|      |               | 优先级 |
| ---- | ------------- | --- |
| 内部样式 | 使用`<style>`标签 | 1   |
| 行内样式 | HTML的stype属性  | 2   |
| 外部样式 |               | 3   |
>推荐：[CSS的引入方式：行内样式表（行内式）、内部样式表（嵌入式）、外部样式表（链接式）](https://blog.csdn.net/qq_44156869/article/details/108569220)

| 样式表   | 优点           | 缺点     | 使用情况 | 控制     |
| ----- | ------------ | ------ | ---- | ------ |
| 行内样式表 | 书写方便 ，权重高    | 结构样式混写 | 较少   | 控制一个标签 |
| 内部样式表 | 部分结构和样式相分离   | 没有彻底分离 | 较多   | 控制一个页面 |
| 外部样式表 | 完全实现结构和样式相分离 | 需要引入   | 最多   | 控制多个页面 |

> [!tip] 提示
> 为了在样式表里覆盖行内声明，需要为声明添加`!important`，这样能将它提升到一个更高优先级的来源。
> 但如果行内样式也被标记为`!important`，就无法覆盖它了。最好是只在样式表内用`!important`。将以上修改撤销，我们来看看更好的方式。
