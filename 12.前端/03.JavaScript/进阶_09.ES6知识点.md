# 11.ES6-函数的剩余参数

函数的剩余参数（Rest Parameters）是JavaScript中用于处理函数调用时**多余参数**的语法特性，它允许将不定数量的参数自动收集到一个数组中，从而简化对多参数的处理。以下是其核心要点：

## 1.定义与语法

剩余参数通过在函数参数前加`...`定义，**必须位于参数列表的最后**。

示例：

```javascript
function example(a, b, ...rest) {
  console.log(rest); // 收集剩余参数为数组
}

example(1, 2, 3, 4); // 输出: [3, 4]
  ```

此时rest是一个包含剩余参数的数组。

## 2.与arguments对象的区别

- arguments是类数组对象，包含所有传入的参数，但无法直接使用数组方法（如map、filter）。
- **剩余参数是真正的数组**，可直接使用数组方法，且仅包含未被显式声明的参数。

```javascript
function compare(a, b, ...rest) {
  console.log(arguments); // 类数组: [1, 2, 3, 4]
  console.log(rest);      // 数组: [3, 4]
}
compare(1, 2, 3, 4);
```

## 3.使用场景

- <font color="#c0504d">当函数需要接受可变数量的参数时</font>，例如数学运算、事件处理等。
- <font color="#c0504d">替代arguments对象</font>，提升代码可读性和安全性（避免依赖隐式对象）。

## 4.注意事项

- 剩余参数**必须是参数列表的最后一个参数**，否则会报错。
- 结合解构赋值或<font color="#c0504d">展开运算符</font>可灵活操作参数。

通过剩余参数，JavaScript提供了更简洁、类型友好的方式处理多参数场景，是ES6的重要改进之一。

# 13.ES6-展开语法

ES6中的展开语法通过`...`符号实现，允许将**可迭代对象**（如数组、字符串）或对象表达式展开为多个元素或键值对。以下是其核心用法及示例：
## 1.函数调用中的展开

可将数组展开为函数参数，替代`apply`方法。

```javascript
const arr = [1, 2, 3];
Math.max(...arr); // 等同于 Math.max(1, 2, 3)
```

## 2.数组操作

1.合并或复制数组

```javascript
const arr1 = [1, 2], arr2 = [3, 4];
const merged = [...arr1, ...arr2]; // [1, 2, 3, 4]
const copy = [...arr1]; // 浅拷贝
```

2.字符串转数组

```javascript
const str = "hello";
const chars = [...str]; // ["h", "e", "l", "l", "o"]
```

## 3.对象展开（ES2018）

将对象属性展开到新对象中，实现合并或部分覆盖：

```javascript
const obj1 = { a: 1 }, obj2 = { b: 2 };
const merged = { ...obj1, ...obj2 }; // { a: 1, b: 2 }
```

后续属性会覆盖前面的同名属性。

## 4.**注意事项**

- **可迭代性**：展开语法要求对象是**可迭代的**（如数组、字符串），普通对象需通过 `Object.keys()` 等方法间接展开。
- **与剩余参数（Rest Parameters）的区别**：展开语法用于“展开”结构，而剩余参数用于“收集”剩余参数为数组：

```javascript
function fn(a, ...rest) { /* rest 是剩余参数 */ }
```

## 典型应用场景

- 简化函数参数传递。
- 合并或克隆数组/对象。
- 动态构建数据结构（如状态管理中的对象更新）。

通过展开语法，代码更简洁且可读性更高，是ES6中常用的特性之一。
