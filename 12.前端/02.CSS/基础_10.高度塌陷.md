# 高度塌陷与BFC

在浮动布局中，**父元素的高度默认是被子元素撑开**的！

```html
<!DOCTYPE html>
<html lang="zh">
    <head>
        <meta charset="UTF-8">
        <title>Title</title>
        <style>
            .outer {
                border: 10px olive solid;
            }

            .inner {
                width: 200px;
                height: 200px;
                background-color: orange;
                /* float: left; */
            }

            .down {
                width: 200px;
                height: 200px;
                background-color: #0a53be;
            }
        </style>
    </head>
    <body>
        <div class="outer">
            <div class="inner"></div>
        </div>
        <div class="down"></div>
    </body>
</html>
```

查看父元素的高度会被撑开，当打开上述`float: left;`注释后，子元素浮动后，其会完全脱离文档流，子元素从文档流中脱离将会无法撑起父元素的高度，导致父元素的高度丢失，父元素高度丢失以后，其下的元素会自动上移，导致页面的布局混乱。

# BFC

## 1.清除浮动解决高度塌陷

### 方案1：设置浮动（不推荐）

```css
.outer {
	border: 10px olive solid;
	float: left;
}

.inner {
	width: 200px;
	height: 200px;
	background-color: orange;
	float: left;
}
```

![[Pasted image 20231025200510.png|400]]

### 方案2：设置为行内块元素（不推荐）

``` css
.outer {
	border: 10px olive solid;
	display: inline-block;
}
```

![[Pasted image 20231025200536.png|400]]

不再独占一行，但是<font color="#f79646">宽度减少</font>，且与下方down之间产生了<font color="#f79646">缝隙</font>。

### 方案3：设置overflow为非visible值

``` css
.outer {
	border: 10px olive solid;
	overflow: hidden;
}
```

使用overflow: auto也可以，overflow: scroll会产生滚动条，不太推荐！

![[Pasted image 20231025200652.png|400]]

可以看到既没有覆盖元素，也保持独占一行的特性（保持了宽度），并且也没有与下方down之间产生缝隙，但是<font color="#f79646">仍然有缺陷</font>。
