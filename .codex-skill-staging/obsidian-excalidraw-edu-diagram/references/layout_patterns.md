# 版式模板

这个 Skill 的首版不做复杂自动排版。Codex 需要先按模板给出每个块的 `x/y/width/height`，渲染器只负责稳定落图。

## 通用规则

- 默认画布：`1600 x 900`
- 上边距：`40`
- 左右边距：`40 ~ 60`
- 主卡片宽度：`260 ~ 340`
- 主卡片高度：`180 ~ 280`
- 两块之间水平间距：`60 ~ 100`
- 两块之间垂直间距：`40 ~ 70`

## 1. 左到右流程图

适用：

- 地址变换
- PV 操作流程
- 计算步骤

推荐：

- 顶部 1 个 `banner`
- 中间一排 `card/callout/table/result`
- 箭头从左到右

坐标建议：

- `banner`: `(40,40,1480,100)`
- 第一列卡片：`x=60, y=220`
- 第二列卡片：`x=420, y=220`
- 第三列卡片：`x=800, y=220`
- 第四列结果：`x=1180, y=220`

## 2. 上下分步图

适用：

- 例题分步解析
- 资源图化简顺序
- 银行家算法安全序列

推荐：

- 顶部 `banner`
- 中间若干层 `card`
- 每层纵向对齐
- 结果放底部 `result`

## 3. 时间轴图

适用：

- 抢占式/非抢占式调度
- 状态变化时序

推荐：

- 时间轴主线在 `y=320 ~ 380`
- 每个时间段一个 `card`
- 关键结论用底部 `result`

## 4. 题干 + 解析卡片图

适用：

- 例题解析
- 概念对比

推荐：

- 左边题干摘要，右边解析
- 或上方题干，下方解析

## 5. 资源分析图

适用：

- 阻塞节点判断
- 资源图化简

推荐：

- 原图摘要或截图说明放左
- 结论卡片放右
- 化简顺序放底部

## 坐标策略

如果用户没有明确要求自由布局，优先按以下策略写规格：

1. 先定 `banner`
2. 再定主卡片行或列
3. 最后补充 `callout/result`
4. 连接线只连块，不直接靠裸坐标画箭头

## 文本策略

- 标题控制在 1~2 行
- 正文尽量按逻辑拆成短句和显式换行
- 不依赖浏览器自动换行去“碰运气”
